## Commands

### Hugo

```bash
hugo server          # serve locally at http://localhost:1313 (live reload)
hugo --minify        # build to public/
```

### Python environment

```bash
source .venv/bin/activate        # activate venv
pip install -r requirements.txt  # restore dependencies
pip freeze > requirements.txt    # update after adding packages
```

### R environment

```bash
R -e "renv::restore()"   # restore R packages from renv.lock
R -e "renv::snapshot()"  # update renv.lock after adding packages
```

## Architecture

Hugo static site using the [Hugo Hero theme](https://github.com/zerostaticthemes/hugo-hero-theme) (inlined as regular files, not a submodule).

```
content/
├── _index.md               # Homepage
├── analysis/               # Services page
├── insights/               # Blog index + articles
│   ├── _index.md
│   └── [article-slug].md
data/                       # YAML data files (team, clients, etc.)
static/                     # Static assets served at root
assets/                     # SCSS / JS processed by Hugo pipes
layouts/                    # Template overrides
themes/hugo-hero-theme/     # Theme source (inlined)
hugo.toml                   # Site config
.github/workflows/deploy.yml  # CI/CD: builds and deploys to GitHub Pages
```

### Deploy workflow

Push to `main` → GitHub Actions builds with Hugo 0.157.0 (extended) → deploys to GitHub Pages.
`public/` and `resources/` are gitignored (generated by CI and local builds).
